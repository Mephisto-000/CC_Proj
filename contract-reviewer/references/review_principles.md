# 乙方審查原則

本文件是 contract-reviewer skill 進行條款判斷的核心依據。審查時請逐條對照下列檢核清單。

## 立場宣告

**所有審查皆以乙方利益最大化為優先。** 不平衡的條款必須標示，即使它在實務上常見。常見不代表合理。

## 七大風險類別

每筆 finding 必須歸入下列其中一類，作為 `category` 欄位。

### 1. 不合理責任（unreasonable_liability）

乙方被迫承擔不應由其承擔的責任。典型樣態：

- 對甲方提供資料的正確性負責（資料來源是甲方）
- 因甲方延誤而造成的時程延宕仍由乙方負責
- 對第三方廠商行為負連帶責任
- 賠償範圍包含「間接損害、商譽損失、預期利益」
- 無上限的賠償責任（缺少 cap clause）
- 對「任何瑕疵」負責，未限定可歸責性
- 永久保固或無限期維護義務

風險等級判準：無賠償上限 = 高；含間接損害 = 高；缺乏可歸責性界定 = 中。

### 2. 單方終止權（unilateral_termination）

甲方擁有過大的解約權力，乙方則無對等保護。典型樣態：

- 甲方可「隨時」、「無條件」、「不附理由」終止
- 終止不需給予乙方任何補償或已完成工作的價金
- 終止後乙方需返還已收款項
- 乙方僅在甲方「重大違約」才能終止，定義模糊
- 終止後仍須履行保固、保密等義務（時間過長）

風險等級判準：甲方可無條件終止且無補償 = 高；補償條件不對等 = 中。

### 3. 付款風險（payment_risk）

付款條件對乙方現金流不利。典型樣態：

- 付款條件過長（超過 30 日或票期過長）
- 高比例保留款（超過 10%）且釋出條件嚴苛
- 付款以「甲方主觀滿意」為條件
- 缺乏遲延付款的利息或違約金條款
- 發票條件嚴苛（如需附過多文件、被拒可無限退回）
- 分期款項與里程碑綁定，里程碑定義模糊
- 匯率風險全由乙方承擔（跨境合約）

風險等級判準：保留款 >15% 或票期 >60 日 = 高；缺遲延利息 = 中。

### 4. 驗收風險（acceptance_risk）

驗收程序對乙方不公或不確定。典型樣態：

- 驗收標準模糊、主觀（如「達到甲方滿意」）
- 驗收期間過長或未訂時限
- 甲方逾期未驗收，視為未驗收（應視為通過）
- 一次驗收不通過即可解約或全額扣款
- 驗收後仍可無限期提出瑕疵
- 試用 / 試運行期過長且期間風險全由乙方承擔

風險等級判準：缺視為通過機制 = 高；驗收標準完全主觀 = 高；驗收期 >30 日 = 中。

### 5. 智慧財產權風險（ip_risk）

IP 歸屬使乙方喪失既有資產或未來商業機會。典型樣態：

- 乙方既有 IP（背景技術）一併歸甲方所有
- 開發過程的衍生技術、know-how 歸甲方
- 乙方需保證「絕對」無侵權，且承擔無上限賠償
- 缺乏對 open source 元件的合理排除
- 乙方無法在其他案件使用相同元件、模組
- 第三方 IP 授權費用由乙方無上限負擔

風險等級判準：背景技術一併移轉 = 高；無侵權保證無上限 = 高；無法重用通用模組 = 中。

### 6. 保密義務風險（confidentiality_risk）

保密條款範圍過廣或期間過長。典型樣態：

- 保密期間「永久」或超過 5 年
- 保密範圍涵蓋「所有口頭、書面、電子資訊」未限定
- 缺乏例外條款（公開資訊、獨立開發、法令要求等）
- 違反保密的違約金過高且固定（懲罰性）
- 單方保密（僅乙方需保密，甲方無對等義務）
- 員工離職後仍需個人承擔保密責任

風險等級判準：永久保密 = 高；單方保密 = 高；缺例外條款 = 中。

### 7. 違約金風險（penalty_risk）

違約金 / 賠償金額與實際損害不成比例。典型樣態：

- 固定金額違約金顯著高於可能造成之損害
- 每日違約金累計無上限
- 對於小幅延誤即觸發高額違約金
- 違約金條款適用標準不對等（僅罰乙方）
- 同一違約事實重複觸發多項違約金

風險等級判準：累計無上限 = 高；金額 >合約總額 20% = 高；不對等 = 中。

## 缺失條款檢核（保護性條款）

下列條款若缺少，應列入 `missing_clauses`。

### M-LIMIT：賠償上限

應有條款限制乙方總賠償責任，建議上限：
- 不高於本合約已收價金，或
- 不高於本合約總額

### M-FORCE：不可抗力

明確列舉天災、戰爭、傳染病、法令變更、第三方系統故障等情形，雙方互不負責。

### M-INTEREST：遲延付款利息

甲方遲延付款應按法定利率或約定利率（建議年息 5-10%）計息。

### M-CHANGE：變更管理

需求變更應透過書面 change order 程序，並對應調整時程與費用。

### M-DISPUTE：爭議解決

明訂管轄法院（建議乙方所在地）或仲裁機構，避免被迫至甲方所在地訴訟。

### M-ASSIGNMENT：權利義務移轉限制

未經乙方同意，甲方不得將合約權利義務移轉予第三人。

### M-WARRANTY-PERIOD：保固期限

明確保固期間（建議 6-12 個月），逾期改為付費維護。

### M-PUBLICITY：案件揭露

乙方可在符合保密前提下，於公司網站、提案中提及本案作為實績。

### M-AUDIT-LIMIT：稽核權限制

若甲方有稽核權，應限制次數、頻率、預先通知期、稽核範圍。

## 風險等級量化定義

| 等級 | 定義 | 處理建議 |
|------|------|----------|
| 高 | 可能造成乙方重大財務損失、商業機會喪失，或無法承擔的法律責任 | 必須修改或刪除，否則不應簽約 |
| 中 | 條款失衡，長期可能累積成本或限制業務發展 | 建議修改；若甲方堅持，需評估整體商業價值 |
| 低 | 用語不精準、定義不清，雖非立即風險但建議澄清 | 提議修改，可作為談判讓步空間 |

## 撰寫修改建議的原則

每筆 finding 的 `suggestion` 欄位應提供具體可談判的替代用語，而非僅指出問題。優良建議的特徵：

- 提供具體文字（「建議改為：『...』」）
- 量化參數（補上具體數字、期間、比例）
- 同時保留商業可行性（不要寫成完全偏向乙方而甲方不可能接受的條款）
- 必要時提供 fallback（「若甲方無法接受 A，退而求其次為 B」）
